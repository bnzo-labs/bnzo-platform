Jailbreak is a browser multiplayer game where the NPCs are Claude agents. Players interact with characters that reason, plan, and respond in natural language. Not scripted. Not decision trees. Actual LLM inference, per tick, per agent.

The architecture question was: how do you run LLM-driven game characters without the game feeling like you're waiting for an API call?

The answer: tool calls. Each agent has a limited set of actions — move, speak, investigate, alert. The LLM picks one per turn. The prompt is tight, the action space is bounded, and token budget caps prevent runaway cost.

Colyseus handles server-authoritative state. PixiJS renders client-side. The agents live on the server. Clients see the result of agent decisions, not the decisions themselves.

The prompt engineering was the real work. Early versions let agents see full room state, which led to leaked information between "competing" agents. The redaction layer — giving each agent a limited view of game state — fixed behavior but added latency. That latency is still the main open problem.

$0.02 per full match at the current token budget. That's viable.
