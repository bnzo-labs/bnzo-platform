jailbreak.live is a browser multiplayer game where the NPCs are Claude agents. Not scripted. Not decision trees. Each agent gets a prompt, a bounded action space, and a per-tick LLM call. Players watch the agents reason through the Boot Chamber — and can interact with them.

The core architecture question was latency. How do you run LLM inference per game tick without the game feeling like it's waiting for an API?

The answer: bounded tool calls. Each agent chooses one action per turn from a small set — move, speak, investigate, alert. The prompt is tight, the context is filtered to what the agent can actually see, and token budget caps prevent runaway cost. Colyseus enforces the game state server-side. PixiJS renders it client-side.

The renderer migration was the first major rewrite. Babylon.js was overkill for a top-down 2D game. PixiJS 7 is fast, has a stable API, and handles the sprite animation needs without the overhead.

The prompt engineering was the real work. Early versions gave agents full room state, which let competing agents implicitly communicate through shared object state. The redaction layer — each agent sees only what's in its line of sight — fixed the behavior but added latency. That's the main open problem.

Incremental movement took the longest to get right. The agent was teleporting to targets instead of moving step by step. Manual update to the movement system fixed it. Continuous movement toward target is now stable.
