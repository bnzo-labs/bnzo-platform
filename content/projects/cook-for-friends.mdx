Cook for Friends started as a question: can an agent replace the group chat chaos that happens before every shared dinner?

The answer is mostly yes. The agent handles menu planning, dietary conflict resolution, and grocery list generation. Hosts share a link. Guests mark preferences. The agent proposes a menu that works for everyone.

The hard part wasn't the LLM. It was the edge cases: vegan + nut allergy + gluten-free in the same group, or guests who don't respond before the planning deadline. We handled both. The grocery API integration didn't make the beta cut.

Cold-start latency on the LLM calls was the other surprise. Users tolerate it once they understand the agent is "thinking," but the first load before any context is cached is rough.

Still in beta. The core loop works. Expanding to larger groups is the next milestone.
