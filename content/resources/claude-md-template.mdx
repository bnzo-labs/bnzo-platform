# CLAUDE.md Template

Every agent-built project needs a CLAUDE.md. It's not documentation — it's the contract between you and every agent that touches the codebase.

## What it is

A structured markdown file that lives at the root of your project. It tells Claude Code (and any subagent you spawn) exactly how to work inside your codebase:

- Mission and stack
- Folder structure with annotations
- Shared library contracts
- API shapes
- Conventions and constraints
- Communication style

Without it, every agent starts cold. With it, agents make decisions that fit your architecture — not generic defaults.

## Why use it

Agent context degrades fast. CLAUDE.md is persistent context that survives session resets, model switches, and team handoffs.

It forces you to write down what you know implicitly. The act of writing it surfaces contradictions in your architecture before agents do.

## What's included

- Root-level CLAUDE.md template with annotated sections
- Conventions block (TypeScript strict, server-first, Tailwind-only, etc.)
- API contracts section
- DB schema section
- Communication style block
- Multi-agent handoff notes

## How to use it

1. Download the template
2. Fill in your stack, mission, and folder structure
3. Add API contracts and DB schema
4. Set conventions that match your codebase
5. Drop in your project root
6. Spin agents — they'll read it automatically

Start lean. Add sections as your codebase grows. The template ships with comments showing what to keep and what to cut.
